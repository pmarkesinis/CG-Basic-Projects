Shading

This assignment focuses on the concepts behind various shading models (which define the appearance of an object’s surface) and the placement of lights (to illuminate these objects). You will learn about concepts such as Lambertian, Phong and Blinn-Phong shading, and you will build your own cartoon-like shader.

In modern graphics pipelines, we would write shaders, which are small programs to define these shading models efficiently, and are often run on the GPU. Today, however, we will work in a simulated framework allowing us to write these shaders on the CPU. The functions you will implement are called for each vertex, and the resulting colour will be mapped on the model. There are many comments in the code to help you along, so read these carefully.

2. Lambertian Model (1 point)

First, we will define a simple diffuse surface shading. The basic formula for the Lambertian model is: \[ D = I K_d (N \cdot L) \]

Here \(I\) is the light colour, \(K_d\) is the diffuse colour, \(N\) is the normal vector at the current point, and \(L\) is the direction from vertex towards the light. Make sure that any light coming from behind the surface results in black \((0,0,0)\).

Implement the Lambertian Model in the diffuseOnly() function in src/your_code_here.h

 

3. Phong Specular Model (1 point)

Second, we will implement the Phong specular model, which introduces view-dependent effects called specularities. The basic formula for the Phong model is: \[ S = I K_s (R \cdot V)^s \]

Here \(I\) is the light colour, \(K_s\) is the specular colour, \(R\) is the reflection vector (light direction reflected away from the normal), \(V\) is the view direction (towards the camera), and \(s\) is the the shininess, influencing the size of the highlight. Be aware: as with the Lambert model a light placed behind the model should not illuminate it from the front!.

Implement the Phong model in the phongSpecularOnly() function in src/your_code_here.h

 

4. Blinn-Phong Specularity Model (1 point)

Third, we will implement the Blinn-Phong specularity model, which is similar to the Phong model above. The basic formula for the Blinn-Phong model is: \[ S = I K_s (H \cdot N)^s \]

The difference here lies in the use of \(N\) (the normal) and \(H\), which is a half vector: a unit vector which lies half way between the view direction \(V\) and the light direction \(L\). Again make sure that a light placed behind the model does not illuminate it from the front.

Implement the Blinn-Phong model in the blinnPhongSpecularOnly() function in src/your_code_here.h

 

5. Gooch Diffuse Shading (1 point)

Now, we will implement Gooch shading, which takes a more stylized approach to shading. Gooch builds on the idea that colour temperature has an effect on how humans perceive depth. Surfaces facing the sun typically appear warmer (yellow) than surfaces facing away from it, which will have a cooler (blue) colour temperature. We model this effect by using the angle between the normal and the light vector (Lambertian) to interpolate between warm and cool: \[ D = I ((\frac{1+N \cdot L}{2}) k_{warm} + (1-\frac{1+N \cdot L}{2}) k_{cool}) \]

The quantity \(N \cdot L\) varies over the interval \([-1,1]\) so a light behind the surface may still illuminate it. The terms \(k_{cool}\) and \(k_{warm}\) mix the surface colour \(k_d\) mixed with a blue/yellow tint respectively: \[ \begin{aligned} k_{cool} &= k_{blue} + \alpha k_d \\ k_{warm} &= k_{yellow} + \beta k_d \end{aligned} \]

where \(k_{blue} = (0,0,b)\) and \(k_{yellow}=(y,y,0)\). \(k_{cool}\) and \(k_{warm}\) may fall outside the valid colour range of \([0, 1]\) which you may ignore. More information on the Gooch shading model can be found in the paper A Non-Photorealistic Lighting Model For Automatic Technical Illustration by Gooch et al.. There is a mistake in one of their formulas, can you spot it?

Implement the Gooch shading model in the gooch() function in src/your_code_here.h

 

And there was light!

Now, we will focus on control of the illumination. For an artist, it can be difficult to control the light positions (just look at the credits of any modern movie, there are hundreds of people working on light placements). Our goal is to simplify this work and provide the artist with a tool to efficiently place lights in an indirect manner. We will use the function userInteraction() in the following exercises. This function receives where the user has “clicked” (as the mouse button is used for the navigation, pressing space will launch this function instead of a click). This interaction will trigger a function userInteraction...(). We will use these functions to define a new light position, based on different criteria.
