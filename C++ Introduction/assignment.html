C++ Exercises

1. Statistics (3 points)

This function is declared as:

std::pair<float, float> statistics(std::span<const float> values);

The input to the function will be a span of valid floating point values (not NaN, +inf or -inf). A span is a collection of items that are contiguous in memory. The output is a pair of values, where the first value is the average of the values in the collection, and the second value is the standard deviation of the values. If the span is empty, it should return 0 for both values. As an example, given the input [-4,-2,0,0,0,2,2,2], the function should return the pair { 0.0f, 2.0f }.

Your implementation should go into your_code_here.h

2. Tree Traversal (3 points)

This function is defined as:

float countTree(const Tree& t, bool countOnlyEvenLevels);

The input to the function will be a tree object and a boolean value defining the behavior of the tree traversal. The Tree structure is defined as:

struct Tree {
    float value;
    std::vector<Tree> children;
};

This struct encodes a tree structure, where each tree can contain other trees as children, and contains a value.

You will need to write the function countTree so that when called, the function will go through the tree and compute the sum of all the values stored in it. If the boolean value countOnlyEvenLevels is set to true, then only the values on even levels of the tree should be included in the sum (the root is at level 0 and considered an even level). You may define any additional struct or function (hint: use recursion).

Your implementation should go into your_code_here.h

Matrix & Vector

Linear algebra is a key building block of computer graphics. In this course we assume that you have basic linear algebra knowledge. If you need a refresher, the TU Delft has made some videos available online. The following exercises are there to help you to familiarize yourself with the basics of matrices and vectors (and C++).

3. Vector Operands (1 point)

In computer graphics we typically operate on 3D worlds hence you’ll see a lot of 3D vectors (in the mathematical sense, not to be confused with the C++ std::vector). We have defined a basic 3D vector class in your_code_here.h:

struct Vector3 {
    float x, y, z;
};

The most common operation we will want to do with a vector is to perform basic mathematical operands: addition, subtraction, element-wise multiplication and element-wise division. You are allowed to ignore division by zero. Please implement the corresponding functions in your_code_here.h:

Vector3 add(const Vector3& lhs, const Vector3& rhs);
Vector3 sub(const Vector3& lhs, const Vector3& rhs);
Vector3 mul(const Vector3& lhs, const Vector3& rhs);
Vector3 div(const Vector3& lhs, const Vector3& rhs);

Apart from these basic operands there are also a couple operations that are very common in computer graphics: computing the dot product, cross product and length of a vector. If you are unfamiliar with any of these operations then please have a look at the math explained videos. Implement the functions for our 3D vector in your_code_here.h.

5. Matrix Operands (0.5 points)

The other key linear algebra construct used in computer graphics are matrices. Matrices are a way to transform vectors in an efficient and unified manner. To describe the operations we desire we need a 3x3 square matrix. In the header file, we have defined the Matrix3 class as follows:

struct Matrix3 {
    Vector3 col1, col2, col3;
};

We reuse the previously defined Vector3 class to make the implementation of the Matrix operations slightly easier. We start, like with Vector3 by declare some basic mathematical operands. Please implement the addition and subtraction functions in your_code_here.h.

Matrix3 add(const Matrix3& lhs, const Matrix3& rhs);
Matrix3 sub(const Matrix3& lhs, const Matrix3& rhs);

6. Matrix Multiplication (1.5 points)

Matrices can describe linear transformations. By multiplying two matrices together we combine the two transformations. Multiplying a vector by a matrix applies the transformations onto the vector. Finally, we can also multiply a matrix by a scalar which means to simply multiply each item in the matrix by said scalar.

We have declared a multiply function for each of these three operations. Implement them in them in your_code_here.h. Keep in mind that a matrix/matrix multiplication is not an element-wise operation.

Matrix3 mul(const Matrix3& lhs, const Matrix3& rhs);
Vector3 mul(const Matrix3& lhs, const Vector3& rhs);
Matrix3 mul(const Matrix3& lhs, float rhs);

7. Matrix Operations (1.5 points)

Apart from the basic operations that you implemented before. We sometimes require more advanced operations such as the transpose, computing the determintant and computing the inverse.

The transpose operation flips the rows and colums. Row 0 becomes column 0, row 1 becomes column 1, etc… The determinant of a matrix describes some properties about the linear operation. It is used as part of the inverse operation. An inverse matrix describes a transformation that is the exact oppposite of the transformation described by the original matrix. You can verify this by computing the inverse and multiplying it by the original matrix. The result should be an identity matrix (1 on the diagonal, 0 everywhere else); i.e. the transformations cancel each other out. You may assume that the input matrix is always invertible.

We recommend the following website for a step-by-step guide on how to invert a 3x3 matrix (and to compute a determinant): https://www.mathsisfun.com/algebra/matrix-inverse-minors-cofactors-adjugate.html

Implement these three functions in your_code_here.h:

Matrix3 transpose(const Matrix3& matrix);
float determinant(const Matrix3& matrix);
Matrix3 inverse(const Matrix3& matrix);
